# Contributing to Distributed Diffusion Model

Thank you for your interest in contributing to this project! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/DistributedDiffusionModel.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `python test_quick.py`
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Follow PEP 8 Python style guide
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular

## Testing

Before submitting a PR, please ensure:

- All existing tests pass: `python test_quick.py`
- Your code doesn't break existing functionality
- New features include appropriate tests

## Pull Request Process

1. Update the README.md if you add new features or change functionality
2. Update the CHANGELOG.md (if applicable) with your changes
3. Ensure your code follows the project's style guidelines
4. Make sure all tests pass
5. Request review from maintainers

## Reporting Issues

When reporting issues, please include:

- Description of the problem
- Steps to reproduce
- Expected behavior
- Actual behavior
- System information (OS, Python version, PyTorch version, GPU info)
- Error messages or logs

## Feature Requests

Feature requests are welcome! Please open an issue with:

- Clear description of the feature
- Use case and motivation
- Proposed implementation (if you have ideas)

## Questions?

Feel free to open an issue for questions or discussions.
